# 3x Weighted Balanced Rank Format

This folder is a 3-pool weighted copy of the 21x balanced version.

The balanced block is still a 256-bit constant-weight block with exactly:

```text
128 ones and 128 zeros
```

The metadata is prefix-coded:

```text
0  -> pool 0
10 -> pool 1
11 -> pool 2
```

## Pool Ratios

The pool fill ratios use `2.72` as the main total ratio:

```text
pool 0 = 1.00 * C(256,128)
pool 1 = 1.00 * C(256,128)
pool 2 = 0.72 * C(256,128)
total  = 2.72 * C(256,128)
```

The code uses exact integer arithmetic:

```text
TOTAL_CAPACITY = floor(C(256,128) * 272 / 100)
POOL_SIZES     = (C256, C256, TOTAL_CAPACITY - 2*C256)
```

Every pool index fits inside the balanced rank capacity, so each `idx` can be converted to one balanced 256-bit block.

## Encode Formula

For a valid input number `R`:

```text
pool 0: 0 <= R < POOL_SIZES[0]
pool 1: POOL_STARTS[1] <= R < POOL_STARTS[1] + POOL_SIZES[1]
pool 2: POOL_STARTS[2] <= R < POOL_STARTS[2] + POOL_SIZES[2]
```

The encoder stores:

```text
metadata prefix + balanced_256(idx)
```

## Decode Formula

The decoder reads the prefix metadata, ranks the balanced block back to `idx`, and rebuilds:

```text
R = POOL_STARTS[m] + idx
```

For files generated by `encode.py`, `decode.py` reconstructs `originals.txt` exactly.
